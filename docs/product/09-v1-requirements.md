# V1 Product Requirements

## V1 Goal

V1 should establish the core financial data model and user-facing workflows needed to make the app useful for real household finance management.

## Must Have

- Linked account support through Plaid or similar
- Manual accounts
- Balance tracking
- Transaction history
- Owner aliases
- Account ownership allocation
- Transaction ownership override
- Historical ownership behavior
- Type/category/subcategory/sub-subcategory model
- Tags
- Rule-based categorization
- Rule priority
- Rule conflict visibility
- Manual transaction overrides
- Transaction splitting
- Net worth dashboard
- Spending dashboard
- Cash flow dashboard
- Budgeting
- Owner-based dashboard filtering
- Local AI assumption
- Authentication scaffolding

## Should Have

- Investment trend dashboard
- Debt payoff dashboard
- Tax tracking
- Mortgage calculator
- Loan calculator
- What-if planning scenarios
- Recurring transaction detection
- Subscription visibility
- AI-generated chart suggestions

## Could Have

- Market analyzer
- Dynamic AI-generated dashboards
- Advanced tax optimization
- Multi-household support
- Hosted LLM provider option
- Mobile-optimized interface

## Explicitly Out of Scope for V1

- Money movement
- Bill payment
- Securities trading
- Commercial SaaS readiness
